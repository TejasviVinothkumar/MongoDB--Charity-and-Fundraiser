EXPERIMENT – 2
(Collection with Required Fields & Data Types)
🔹 Aim

To enforce required fields using schema validation.

🔹 Query
db.createCollection("campaigns", {
  validator: {
    $jsonSchema: {
      bsonType: "object",
      required: ["campaignTitle", "campaignId", "organizationName"],
      properties: {
        campaignTitle:    { bsonType: "string" },
        campaignId:       { bsonType: "string" },
        organizationName: { bsonType: "string" },
        goalAmount:       { bsonType: ["double", "null"] },
        isActive:         { bsonType: "bool" }
      }
    }
  }
})

🔹 Valid Insert
db.campaigns.insertOne({
  campaignTitle:    "Clean Water for All",
  campaignId:       "CMP2025001",
  organizationName: "AquaHope NGO",
  goalAmount:       500000.5,
  isActive:         true
})

🔹 Invalid Insert (Missing field)
db.campaigns.insertOne({
  campaignTitle:    "Feed the Children",
  organizationName: "HopeFoundation"
})

🔹 Result

The collection enforces required fields. Invalid documents are rejected.

EXPERIMENT – 3
(Advanced Validation – Enum, Range, Pattern)
🔹 Aim

To apply advanced validation rules.

🔹 Query

db.createCollection("donations", {
  validator: {
    $jsonSchema: {
      bsonType: "object",
      required: ["donorName", "donationCategory", "amount", "unitsReceived"],
      properties: {
        donorName: {
          bsonType: "string",
          minLength: 3,
          maxLength: 100
        },
        donationCategory: {
          enum: ["Food & Water", "Medical Aid", "Education",
                 "Disaster Relief", "Animal Welfare"]
        },
        amount: {
          bsonType: "double",
          minimum: 50,
          exclusiveMinimum: true
        },
        unitsReceived: {
          bsonType: "int",
          minimum: 1
        },
        transactionId: {
          bsonType: ["string", "null"],
          pattern: "^TXN[0-9]{6}[A-Z]{2}$"
        }
      }
    }
  },
  validationLevel: "strict",
  validationAction: "error"
})

🔹 Valid Insert
db.campaigns.insertOne({
  campaignTitle:    "Clean Water for All",
  campaignId:       "CMP2025001",
  organizationName: "AquaHope NGO",
  goalAmount:       500000.1,
  isActive:         true
})

🔹 Invalid Insert
db.donations.insertOne({
  donorName:        "Arjun Mehta",
  donationCategory: "Sports",
  amount:           1000.1,
  unitsReceived:    NumberInt(5)
})

🔹 Result

Advanced validation ensures correct donor data and prevents invalid entries.

EXPERIMENT – 4
(Modify Validation on Existing Collection)
🔹 Aim

To add validation rules to an existing collection.

🔹 Queries
db.createCollection("volunteers")
db.runCommand({
  collMod: "volunteers",
  validator: {
    $jsonSchema: {
      bsonType: "object",
      required: ["volunteerId", "fullName", "role", "hoursCommitted"],
      properties: {
        volunteerId: {
          bsonType: "string",
          pattern: "^VOL[0-9]{5}$"
        },
        fullName: {
          bsonType: "string",
          minLength: 3
        },
        role: {
          enum: ["Field Worker", "Coordinator",
                 "Fundraiser", "Medical Staff"]
        },
        hoursCommitted: {
          bsonType: "double",
          minimum: 1.0
        }
      }
    }
  },
  validationLevel: "moderate"
})

🔹 Result

Validation rules were successfully added to the existing collection.
